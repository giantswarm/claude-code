## Labels

**Required:**
- `team/<name>` - Team to evaluate/own
- `kind/request`

**Recommended:**
- `customer/<customer-name>` - If customer-specific
- `function/<area>` - For cross-functional evaluation
- `priority/<level>` - If priority is clear

## Examples

Good request titles:
- `Customer Request: Enable SCIM provisioning for PagerDuty integration`
- `Internal Request: Add automated cluster backup scheduling to UI`
- `ACME Corp: Support for custom CNI plugins in workload clusters`

## Quality Checklist

- [ ] Clear requestor identification
- [ ] Use case explained
- [ ] Business value articulated
- [ ] Contact information provided
- [ ] Next steps for evaluation defined
